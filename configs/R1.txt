🔹 Router R1 (Core Services & Routing)

Role:

Inter-VLAN routing (router-on-a-stick)
DHCP server for VLANs
OSPF routing
Security enforcement (ACL + SSH)

Key Configuration:

! Inter-VLAN Routing
interface g0/0/0.10
 encapsulation dot1Q 10
 ip address 192.168.10.1 255.255.255.0

interface g0/0/0.20
 encapsulation dot1Q 20
 ip address 192.168.20.1 255.255.255.0

interface g0/0/0.30
 encapsulation dot1Q 30
 ip address 192.168.30.1 255.255.255.0

👉 Enables communication between VLANs using subinterfaces.

! DHCP Configuration
ip dhcp pool VLAN10
 network 192.168.10.0 255.255.255.0
 default-router 192.168.10.1

👉 Automatically assigns IP addresses to hosts in each VLAN.

! OSPF Routing
router ospf 1
 network 192.168.10.0 0.0.0.255 area 0
 network 10.0.0.0 0.0.0.3 area 0

👉 Allows dynamic routing between R1 and R2.

! ACL (Traffic Control)
access-list 100 deny ip 192.168.10.0 0.0.0.255 192.168.20.0 0.0.0.255
access-list 100 permit ip any any

👉 Blocks HR (VLAN 10) from accessing IT (VLAN 20).

! SSH Security (Management VLAN Only)
access-list 10 permit 192.168.99.0 0.0.0.255

line vty 0 4
 access-class 10 in
 transport input ssh

👉 Restricts device access to management VLAN only.
