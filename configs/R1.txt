========================
R1 CONFIGURATION
========================

Hostname:
- Identifies the device as R1 in the network
hostname R1

----------------------------------------
DHCP Configuration
----------------------------------------
- Provides automatic IP addressing for VLANs 10, 20, and 30
- Excludes gateway addresses from assignment

ip dhcp excluded-address 192.168.10.1
ip dhcp excluded-address 192.168.20.1
ip dhcp excluded-address 192.168.30.1

ip dhcp pool VLAN10
 network 192.168.10.0 255.255.255.0
 default-router 192.168.10.1
 dns-server 8.8.8.8

ip dhcp pool VLAN20
 network 192.168.20.0 255.255.255.0
 default-router 192.168.20.1
 dns-server 8.8.8.8

ip dhcp pool VLAN30
 network 192.168.30.0 255.255.255.0
 default-router 192.168.30.1
 dns-server 8.8.8.8

----------------------------------------
Inter-VLAN Routing (Router-on-a-Stick)
----------------------------------------
- Uses subinterfaces to route between VLANs

interface g0/0/0
 no ip address

interface g0/0/0.10
 encapsulation dot1Q 10
 ip address 192.168.10.1 255.255.255.0
 ip access-group 100 in

interface g0/0/0.20
 encapsulation dot1Q 20
 ip address 192.168.20.1 255.255.255.0

interface g0/0/0.30
 encapsulation dot1Q 30
 ip address 192.168.30.1 255.255.255.0

interface g0/0/0.99
 encapsulation dot1Q 99
 ip address 192.168.99.1 255.255.255.0

----------------------------------------
Point-to-Point Link (R1 ↔ R2)
----------------------------------------
- Used for OSPF routing between routers

interface g0/0/1
 ip address 10.0.0.1 255.255.255.252

----------------------------------------
OSPF Routing
----------------------------------------
- Dynamically shares routes between routers

router ospf 1
 network 192.168.10.0 0.0.0.255 area 0
 network 192.168.20.0 0.0.0.255 area 0
 network 192.168.30.0 0.0.0.255 area 0
 network 10.0.0.0 0.0.0.3 area 0

----------------------------------------
ACL (Traffic Segmentation)
----------------------------------------
- Blocks VLAN 10 from accessing VLAN 20

access-list 100 deny ip 192.168.10.0 0.0.0.255 192.168.20.0 0.0.0.255
access-list 100 permit ip any any

----------------------------------------
SSH Configuration
----------------------------------------
- Enables secure remote access

ip domain-name lab.local
ip ssh version 2
username admin secret <encrypted>

----------------------------------------
Management Access Control
----------------------------------------
- Only allows VLAN 99 to SSH into device

access-list 10 permit 192.168.99.0 0.0.0.255

line vty 0 4
 access-class 10 in
 login local
 transport input ssh
